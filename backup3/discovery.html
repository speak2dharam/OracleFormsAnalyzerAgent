Discovery Architecture
6-layer pipeline — data flows bottom-up: Collection → Parsing → Analysis → Storage → API → Dashboard
L1: Collection
Discovery Agents  ·  Agentless Scanners  ·  File Upload
Connects to Oracle instances, OS, middleware via WMI/SSH/JDBC — no agent install required for agentless mode
Python + PowerShell + JDBC + SSH
L2: Parsing
AWR Parser  ·  FMB Binary Parser  ·  DDL Parser
Extracts structured metadata from binary Oracle Forms files, AWR HTML reports, DDL exports, RDF report definitions
Python + Node.js + ANTLR4
L3: Analysis
Dependency Mapper  ·  AI Complexity Scorer
Builds cross-domain dependency graph, scores PL/SQL & Forms complexity 1–10 using Azure OpenAI
.NET Core + Neo4j + Azure OpenAI
L4: Storage
Metadata Repository
Persists all discovery artefacts, object inventories, complexity scores, and data lineage graphs
Azure SQL + Cosmos DB
L5: API
Discovery REST API
Serves structured discovery data to downstream assessment, TCO, wave planning, and modernization modules
.NET Core 8
L6: Dashboard
Estate Explorer  ·  Dependency Viewer
Interactive workload browser, cross-domain dependency graph explorer, and per-domain drill-down views
React + TypeScript
