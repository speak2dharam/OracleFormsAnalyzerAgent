Recommended Migration Roadmap
ODB@Azure Exascale BYOL + Oracle Forms 12c lift and shift to Azure VM
Recommended Path

Migrate HRDPRD database to ODB@Azure Exadata Exascale while lifting Oracle Forms 12c + WebLogic 12c to a D8ds_v5 Azure VM. Preserve RAC, eliminate on-prem infrastructure, integrate Entra ID SSO, and evaluate APEX modernization in Year 2-3.

1
Planning and License Setup - Weeks 1-3

Confirm 32 Oracle EE+RAC processor licenses and 8 WLS Suite processor licenses. Activate ODB@Azure via MACC and Support Rewards.

2
Provision Exascale DB Cluster - Weeks 3-6

Provision 32 ECPUs across two nodes, configure Oracle Net, Data Guard, listeners, wallet, and TDE keys.

3
Provision Forms VM - Weeks 4-8

Deploy D8ds_v5 VM, install Forms 12c + WLS 12c BYOL, configure Entra ID LDAP, and connect Forms to Exascale.

4
Data Migration - Weeks 6-10

Run RMAN initial load and GoldenGate CDC with schema freeze and DBMS_COMPARISON validation.

5
Test and Validate - Weeks 9-13

Execute HR/payroll batch testing, Forms regression, DR drill, SSO validation, and 150-user load test.

6
Production Cutover + Hypercare - Weeks 13-14

Switch TNS, Forms app config, and DNS. Run 30-day hypercare and decommission on-prem after 60-day parallel run.

Assumptions: HRDPRD AWR 35-day capture, 2 RAC nodes, Forms 12c/WebLogic 12c, peak 150 concurrent Forms users, existing 32 EE+RAC and 8 WLS Suite processor licenses, ODB@Azure Exascale BYOL ECPU rate $1.77/hr, Support Rewards at 25% of ECPU spend, WLS support 8 proc x $100K x 22% = $176K/yr.
