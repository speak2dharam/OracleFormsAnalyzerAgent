Meeting-ready internal script

This updated clickthrough positions the solution as an Oracle modernization platform, not just a parser or report viewer. We start with upload plus future discovery-source connectors. Then we normalize forms into canonical JSON, apply AI analysis, surface key observations, map dependencies, and run assessment with a retain-versus-modernize decision. The DEMODB reports are no longer buried in PDF output. Their object-level effort, compatibility issues, and hotspots now show up in a schema drill-down screen. Conversion now defaults to React + .NET + PostgreSQL, with Angular still available when needed. Most importantly, generated output goes through Review & Validate before deployment. TCO is now part of the story: phased modernization is the lowest 5-year cost option, while Oracle on Azure remains a faster but more expensive retain path.
