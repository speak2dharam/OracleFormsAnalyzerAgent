Section 10 · Risk Analysis
Key Risks & Mitigations
Migration risks identified across database, Oracle Forms, licensing, and operational dimensions — with specific mitigations for the recommended ODB@Azure Exascale path.
TX Row Lock Application Bug
HIGH · All options
enq: TX row lock is 11.47% of DB time — this is an application-level issue that persists regardless of migration option. Will not be resolved by moving to Exascale or PostgreSQL without application remediation.
→ Engage app team to identify root cause before migration. Consider enabling lock diagnostics and ADDM-guided investigation in Year 1.
Forms 12c EOL Risk
MEDIUM · Sustaining to 2028
Oracle Forms 12c Premier Support ended December 2024. Only sustaining support (security patches, no new features) runs to December 2028. Post-2028 without upgrade = unsupported production workload.
→ Plan Forms 14c upgrade or APEX migration by end of 2027. Recommended: APEX in Year 2 to eliminate WLS license and extend support horizon.
Oracle BYOL License Compliance
LOW · with planning
Running Oracle DB and WLS on Azure VMs requires compliance with Oracle's Azure BYOL terms. WLS rule: 2 vCPU = 1 processor license. DB EE+RAC: 2 vCPU = 1 processor license on Azure VMs.
→ Confirm all BYOL terms with Oracle Account Team before provisioning. ODB@Azure uses ECPU-based billing — no core factor confusion.
Network Latency (DB↔Forms)
LOW · with VNet design
Oracle Forms uses stateful JDBC/SQL*Net connections to the database. On-prem latency is typically <1ms. ODB@Azure + Forms VM on Azure in the same region should achieve <2ms — within acceptable range for Forms.
→ Deploy Forms VM and ODB@Azure in the same Azure region (East US 2). Use VNet peering. Validate <2ms latency as Phase 2 exit criteria.
Payroll Peak Sizing
MEDIUM · needs validation
AWR shows avg AAS of 4.05 but peak AAS of ~80 during payroll runs. The recommended 32 ECPU configuration must handle this 20x peak. Elastic ECPU scaling can help, but burst behaviour needs testing.
→ Run payroll cycle simulation in Exascale test environment before cutover. Configure ECPU auto-scale threshold. Retain ability to temporarily boost ECPUs during peak.
GoldenGate CDC Data Sync Risk
LOW · managed process
During migration, GoldenGate Change Data Capture synchronizes from on-prem Oracle 12.2 to Exascale. Schema changes during CDC window could cause replication lag or errors.
→ Enforce schema change freeze from Phase 4 start. Monitor GoldenGate lag continuously. Validate with DBMS_COMPARISON. Target CDC lag <5 seconds before cutover.
WLS / Entra ID Integration
LOW · documented path
Configuring WebLogic Server LDAP to authenticate against Azure Entra ID requires LDAP provider configuration in WLS Domain. Oracle provides official documentation for Azure AD integration.
→ Configure WLS LDAP provider to point to Azure Entra ID LDAP endpoint. Test SSO login in Phase 3 before production. Maintain fallback local admin account during transition.
DBA Knowledge Transfer
MEDIUM · mitigable
Moving to ODB@Azure shifts some DBA responsibilities to Oracle managed service. Internal DBAs need familiarity with OCI console, ECPU scaling, ARS backup management, and Support Rewards tracking.
→ Enroll DBAs in Oracle Cloud Infrastructure training (OCI DBA certification). Engage Oracle or certified partner for 3-month hypercare period post go-live.

License Validation & Oracle Account Engagement

Subscribe ODB@Azure via Azure Marketplace (MACC)

Wks 1–2Cloud Arch

VNet Architecture Design & NSG Rules

Design VNet with dedicated subnets: ODB@Azure subnet, Forms VM subnet. Configure VNet peering. Set NSG rules for SQL*Net (port 1521) between Forms and DB. Target <2ms DB↔App latency.

Wks 2–3Network + DBA
4

Provision Exascale DB Cluster (32 ECPUs, 2-node HA)

Provision Exascale VM cluster with 32 ECPUs (16 per node). Configure Oracle Net, listeners, Data Guard standby. Migrate Oracle Wallet/TDE. Validate latency and connectivity from planned Forms subnet.

Wks 3–6DBA Team
5

Deploy Forms VM (D8ds_v5) + WLS + Forms 12c BYOL

Deploy D8ds_v5 Azure VM in East US 2. Install Oracle Linux, Oracle WebLogic 12c + Forms 12c BYOL. Configure WLS domain. Integrate LDAP with Azure Entra ID. Test Forms→DB connectivity and SSO login.

Wks 4–8App/WLS Admin
6

Execute Database Migration (RMAN + GoldenGate CDC)

RMAN initial load to Exascale. Enable GoldenGate CDC for live sync. DBMS_COMPARISON validation. Optional: upgrade Oracle 12c → 19c during migration window. Enforce schema change freeze from this point.

Wks 6–10DBA + Data Eng
7

Full Regression Testing — DB + Forms End-to-End

DB regression: HR/payroll batch jobs, all PL/SQL packages, batch scheduling. Forms regression: all screens end-to-end, all user roles. Performance vs on-prem baseline. DR failover drill for Data Guard. Load test 150 concurrent Forms users.

Production Cutover — DB + Forms Simultaneously

DB: update TNS alias to Exascale. Forms: update frmservlet config and DNS/URL. 30-day hypercare with on-prem standby. Decommission on-prem hardware after 60-day parallel. Confirm first Support Rewards credit.
