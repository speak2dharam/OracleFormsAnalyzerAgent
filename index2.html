8. Technology Decision

This product should present .NET + Angular as the main modernization target. Other technologies should appear as optional support, not the default story.

Default stack

Frontend: Angular
Strong enterprise forms, routing, validation, componentization, and structured UI migration.
Backend: .NET Web API
Good for service layering, DTOs, auth, reporting APIs, and enterprise integration.
Database: PostgreSQL
Target cloud-native DB for Oracle exit strategy.

Optional support

Python
Best suited for parser engine, XML transformation, dependency extraction, report generation, and AI orchestration.
Java
Use only if customer environment or integration requirements clearly justify it.
React
Can remain an optional output target, but Angular should be the main story here.
ready message
We should position .NET and Angular as the standard modernization path because it is clearer, more controllable, and better aligned with enterprise migration delivery.

Why this positioning helps

Cleaner go-to-market
One clear target architecture is easier to sell than too many framework choices.
Faster delivery
Reusable templates and patterns can be built around one primary stack.
Better demo story
Assessment → Angular UI → .NET API → PostgreSQL is easy to explain.
Optional flexibility still exists
Python remains useful behind the scenes; Java stays optional for special cases.

Meeting-Ready Script

This project is not only a Forms parser. It is an Oracle modernization hub. First, we ingest Oracle Forms artifacts and convert them into a canonical JSON model. Then AI helps explain triggers, business rules, and dependencies. On top of that, we attach Azure assessment output to show likely PostgreSQL target costs and service recommendations. The next important layer is recurring post-migration value. We can show security controls, threat visibility, cost optimization actions, FinOps recommendations, and drift detection. That means the product story does not stop at migration. For modernization output, we should make .NET Web API and Angular our standard target stack. Python remains useful for parser and report automation, while Java should stay optional for customer-specific requirements.
